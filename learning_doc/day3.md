# Day 3: 上下文与记忆机制

## 目标
掌握 nanobot 的会话管理 (Session) 与记忆截断算法 (Memory)。

## 学习任务

1. **探索 Session 包 (`nanobot/session/`)**
   - 对话的核心持久化是如何存储的？了解系统在磁盘 `~/.nanobot/sessions/` 的历史 JSON 文件记录格式。
   - 会话管理器是如何做到跨平台隔离的 (例如不同的 Telegram 群聊、不同的个人微信能彼此不干扰)。

2. **分析 Memory 算法 (`nanobot/agent/memory.py`)**
   - 当对话超过一定的轮数，或者积累了太多的上下文后，如何处理限制？
   - 阅读并理解 `truncate_xxx` 或类似的截断方法：它是按照轮数粗暴删除，还是利用 Token 长度计算滑动窗口的？
   - 阅读多模态消息 (图片/文件) 特殊情况下的截断保留策略。

3. **深入理解 `System Prompt`**
   - 在 `nanobot/templates/` 中找到默认系统提示词，理解人设设定和当前时间/环境信息的注入原理。

## 代码实验
- 人为修改一次你的 Session 记录 JSON (可以去系统生成目录找找)，把过去某句话替换成其他的。然后发起对话，测试大模型是否真的基于你篡改的历史进行回答。

## 思考题
- 为什么 Token 的估算对记忆管理如此重要？如果计算出错了会有什么后果？
- System Prompt 为什么要和最近几轮的用户消息混合，而不是每次都全量带上所有从第一天开始聊的消息？
