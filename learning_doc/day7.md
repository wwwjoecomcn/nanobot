# Day 7: 定时任务、路由及其余高级特性

## 目标
学习框架中作为中枢调度与运维的关键组件，理解工程的完整生命周期。

## 学习任务

1. **命令路由体系 (`nanobot/command/`)**
   - 当你在各端里发送 `/status`、`/restart` 或别的斜杠控制指令时发生了什么？
   - 控制层是如何将这种命令拦截下来，并且不在内部消耗大模型的额度，而且直接返回硬件指标等结果的？

2. **Crontab 定时器 (`nanobot/cron/`)**
   - 用户是如何吩咐“每天早上8点推送最新早报并搜索新闻”这种自然语言调度的？
   - 剖析 Cron 这个模块如何与全局 Agent 生命周期挂钩。

3. **心跳安全与隔离 (`nanobot/heartbeat/` & `nanobot/security/`)**
   - 当模型工具执行卡死产生脏状态时，虚拟工具心跳兜底是如何在循环中重置并退出死锁进程的？
   - `security.py` 或安全文档 `SECURITY.md` 提出了什么级别的隔离约束？例如，在允许访问本机哪些文件夹、以及读取多大的文件时是如何进行软边界控制的？

4. **生产部署**
   - 阅读 `Dockerfile` 与 `docker-compose.yml`，清楚如何在服务器实现无挂断运行。
   - 阅读 Systemd Linux 服务文件示例 (如果存在)，学习在 VPS 后台运行 `nanobot gateway` 与自启兜底的最佳实践。

## 毕业测验/思考题
- 如果要求你将 nanobot 对接到一家本地私有的医院 OA 系统，你需要改写/添加该项目的几个核心组件？
- 回望 7 天，该轻量架构的设计哲学与你看到过的别的 “巨型厚重 LangChain/AutoGPT 类似项目”比起来，最大的取舍点是什么？
